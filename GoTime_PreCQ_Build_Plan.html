Risks & Mitigations

#RiskSeverityMitigation
1 BAA not executed before PHI enters system CRITICAL Begin BAA pursuit Week 1. Block all PHI until confirmed. Synthetic data only in dev/test.
2 Staff resistance to structured intake form HIGH Interview intake staff before build. Auto-populate linked fields. Quick-add modals. UAT with real staff.
3 Supabase RLS misconfiguration exposes PHI HIGH RLS policy audit in Phase 5. Automated role boundary tests. HIPAA-focused review.
4 n8n instance outage blocks notifications MEDIUM Use n8n Cloud SLA or deploy with redundancy. Retry logic on all webhooks. Python API fallback direct to Twilio/SendGrid.
5 Data quality issues delay CQ migration MEDIUM Enforce structured dropdowns from day one. Monthly data quality reviews. Stable UUIDs on all records.
